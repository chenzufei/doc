
1、python的垃圾回收机制（gc，Garbage Collection）：
（1）引用计数：容器存在循环引用时，引用计数无法归零，从而无法回收。 
（2）标记-清除：通过三色标记方法，修正引用计数。
                方案：广度遍历root链表中的容器对象，通过三色标记(Tri-color Marking，白色，灰色，黑色)，
                最终将不可达的的对象标记为黑色（引用计数减为0）。
（3）分代回收：采用空间换时间的策略，目的是减少垃圾检测带来的额外开销；
               实现方式：三条可收集对象链表


备注：
（1）参考见《源码探秘 CPython》
（2）只有容器（列表、字典、类、元组）才会出现循环引用，整数、字符串、浮点数等不会。


任何对象创建都会加入到refchain
container对象还会加入可收集对象链表
